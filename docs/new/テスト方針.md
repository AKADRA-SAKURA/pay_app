# テスト方針（pay_app）
- 最終更新日: 2026-02-18
- 対象プロジェクト: `pay_app`

## 1. 目的
機能追加・改修時の回帰を抑止し、資金予測結果の信頼性を維持する。

## 2. テストレベル
- UT（単体）: 日付計算、集計関数、パーサ等の純ロジック
- IT（結合）: API + DB + スケジューラ連携
- ST（システム）: 主要業務シナリオのE2E確認
- UAT（受入）: 利用者観点での操作/表示確認

## 3. 対象機能
- 口座/予定/サブスク CRUD
- カード明細・リボ・分割・チャージ計上
- イベント再作成
- 予測（マイナス警告含む）
- 円グラフ集計（支払内訳、カード店舗、年間コスト）

## 4. 実施ルール
- PR前に最低限 `unittest` を実行
- 仕様変更時は該当UTを追加
- 破壊的変更はIT/STで手動確認を必須

## 5. 完了条件
- 重大不具合（S1/S2）が0件
- テスト観点に対し未実施がない
- 変更箇所に対応するテスト結果が記録済み

## 6. 不具合管理
- 不具合は `docs/test/不具合管理.md` に記録
- 再現手順と修正確認手順を必須記載
