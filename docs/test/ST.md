# ST（システムテスト）
- 最終更新日: 2026-02-18
- 対象プロジェクト: `pay_app`

## 1. 目的
実運用に近い操作で、画面全体の業務フローが成立することを確認する。

## 2. テストシナリオ
1. 初期設定
- 口座2件、カード1件を登録
- 有効開始日必須入力を確認

2. 月次運用
- 給料（収入）と定期支払（予定）を登録
- サブスクを複数頻度で登録
- イベント再作成し、当月/翌月表示を確認

3. カード運用
- 明細を手入力/CSV/テキストで登録
- リボ/分割/チャージを追加
- カード請求額・店舗割合円グラフを確認

4. 予測確認
- 総資産がマイナス時に警告表示されること
- 口座有効期間の開始/終了境界が反映されること

5. 一括操作
- 一括編集→一括保存
- 一括選択削除

## 3. 判定
- 各シナリオで致命的エラーなし
- データの欠損/二重計上なし
