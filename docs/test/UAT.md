# UAT（受入テスト）
- 最終更新日: 2026-02-18
- 対象プロジェクト: `pay_app`

## 1. 目的
利用者視点で「日常的に資金繰り管理ができるか」を確認する。

## 2. 受入観点
1. 入力しやすさ
- 必須項目が分かる
- 編集・削除・一括操作が直感的

2. 見やすさ
- 月次の予定・請求・予測が追える
- 円グラフで比率が把握できる

3. 信頼性
- イベント再作成後に金額が整合する
- リボ/分割/チャージの反映漏れがない

4. 運用継続性
- 明細取込失敗時の修正が容易
- 不具合時の再現手順が残せる

## 3. 受入判定
- 主要ユースケースで業務継続可能
- 重大な誤計上・データ欠損がない
- 利用者承認を得られること
